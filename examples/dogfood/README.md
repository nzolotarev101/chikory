# Dogfood task specs

One file per dogfood campaign тАФ the TaskSpec that ran (or will run) a
plan.md work-package slice through Chikory itself. Full operating manual:
[`docs/DOGFOODING.md`](../../docs/DOGFOODING.md).

## Convention

- **Path**: `examples/dogfood/dogfood-<NNN>.yaml` тАФ `NNN` is the campaign
  number, allocated sequentially. The paired report lives at
  `docs/reports/dogfood-<NNN>.md` (same number; the report is mandatory тАФ
  TASK-PROTOCOL ┬з7).
- **`name:` field**: `dogfood-<NNN>-<short-slug>` (e.g.
  `dogfood-001-memory-pointer-store`).
- **Header comment**: which WP/slice it implements, the launch command, and
  тАФ once it has run тАФ the run-id, outcome, landed commit, and report link.
- Specs are kept after the run (they're the reproducible half of the
  report); never rewrite the `goal`/criteria of a spec that already ran тАФ
  a new attempt or next slice is a new `dogfood-<NNN+1>.yaml`.
- After every terminal run: `/dogfood-review <run-id>` тАФ the standardized
  review that verifies the delivery, writes the report, updates the plan,
  and readies the next spec (DOGFOODING.md ┬з6.1).
